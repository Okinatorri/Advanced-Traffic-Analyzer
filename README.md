# Advanced-Traffic-Analyzer

Скрипт `advanced_traffic_analyzer.py` призначений для аналізу логів веб-сервера у форматі:

<timestamp> <ip_address> <http_method> <url> <status_code> <response_size>

makefile
Копировать код

Наприклад:

1717020800 192.168.1.10 GET /home 200 1500
1717020900 192.168.1.11 POST /api/login 401 120
1717021000 192.168.1.10 GET /profile 200 800

markdown
Копировать код

Скрипт підтримує:

- Фільтрацію за HTTP-методом (`--method`)
- Фільтрацію за статус-кодом або діапазоном (`--status`, наприклад `400-499`)
- Фільтрацію за часовим інтервалом (`--start` та `--end`, Unix timestamp)
- Вивід топ-N активних IP (`--top`, за замовчуванням 3)

Виводиться статистика:

- Загальна кількість запитів після застосування фільтрів
- Кількість унікальних IP
- Топ-N активних IP
- Кількість унікальних IP за останні 24 години від останнього запису
- Розподіл за HTTP-методами (у %)
- Топ-5 найчастіше запитуваних URL
- Загальний обсяг переданих даних (з можливістю читабельного формату)
- Кількість помилкових запитів (4xx та 5xx)
- Середній розмір відповіді для успішних запитів (2xx)
- Запити за години останніх 24 годин

Скрипт коректно обробляє помилки:

- Відсутність файлу або відсутність прав на читання
- Некоректний формат рядків (виводить попередження у stderr)
- Некоректні аргументи командного рядка

Алгоритмічна складність
Читання та парсинг файлу: O(N), де N - кількість рядків у лог-файлі

Збирання статистики: O(N) завдяки використанню Counter та set

Вивід топ-N IP: O(M log N), де M — кількість унікальних IP (використовується most_common у Counter)

Всі операції відбуваються лінійно за кількістю рядків з мінімальним використанням пам'яті

Скрипт оптимізований для файлів до ~1 000 000 рядків.

Можливі покращення
Паралельна обробка великих файлів для підвищення швидкості

Підтримка gzip-сжатих лог-файлів

Більш детальна статистика по URL-ах та статус-кодах

Збереження результату у JSON або CSV для подальшої обробки

Візуалізація запитів по годинах за допомогою графіків
